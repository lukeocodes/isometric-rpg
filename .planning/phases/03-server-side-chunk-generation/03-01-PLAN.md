      Layer 2: biome-specific fBm noise using BIOME_TERRAIN_PROFILES (remapped to [0,1] per octave, multiplied by amplitude)

    - terrain-noise: BIOME_TERRAIN_PROFILES enhanced from client version -- mountains get higher amplitude (3.5), snow peaks get extreme amplitude (4.5) and frequency (0.14) so they are consistently impassable walls.     - chunk-generator: generateChunkHeights(chunkX, chunkZ, worldMap, perm) returns a 2048-byte Buffer containing 1024 Float16 values (32x32 grid)
    - chunk-generator: Heights are written as little-endian Float16 via DataView.setFloat16()
    - chunk-generator: Continental elevation read from worldMap.elevation[chunkZ * worldMap.width + chunkX]
    - chunk-generator: Biome ID read from worldMap.biomeMap[chunkZ * worldMap.width + chunkX]
    - chunk-generator: Same inputs always produce bit-identical output (deterministic)

    Create `packages/server/src/world/chunk-cache.ts`:
    1. Import { redis } from "../db/redis.js", zlib from "node:zlib", types, constants.
    2. Import { generateChunkHeights } from "./chunk-generator.js".

    Binary serialization:
    3. `export function computeElevationBands(elevation: Float32Array): Uint8Array`:
       - Allocate Uint8Array of same length
       - For each value, quantize to 0-6 using thresholds [0.15, 0.30, 0.45, 0.60, 0.75, 0.90, 1.01]
       - Return Uint8Array

    4. `export function serializeWorldMap(worldMap: WorldMap, seed: number): Buffer`:
       - Compute elevationBands from worldMap.elevation via computeElevationBands()
       - Extract regionBiomes: create Uint8Array(worldMap.regions.length), fill with region.biome values
       - Header: 4(magic) + 4(seed) + 2(width) + 2(height) + 4*4(lengths) = 28 bytes
       - Write magic 0x574D4150, seed, width, height, then 4 payload lengths
       - Copy biomeMap, elevationBands, regionMap (as raw bytes), regionBiomes into buffer after header
       - Return Buffer

    5. `export function deserializeWorldMap(buf: Buffer): { seed: number; width: number; height: number; biomeMap: Uint8Array; elevationBands: Uint8Array; regionMap: Uint16Array; regionBiomes: Uint8Array }`:
       - Read header, validate magic === 0x574D4150
       - Extract arrays at computed offsets
       - For regionMap: use `new Uint16Array(buf.buffer.slice(offset, offset + len))` to handle alignment
       - Return parsed object

    6. `export function gzipWorldMap(worldMap: WorldMap, seed: number): Buffer`:
       - Call serializeWorldMap, then zlib.gzipSync() on result
       - Return gzipped buffer

    Redis caching:
    7. `export async function cacheWorldMap(seed: number, gzippedBuf: Buffer): Promise<void>`:
       - `await redis.setBuffer(\`worldmap:seed:${seed}\`, gzippedBuf)`
       - No TTL -- persists until seed changes (different key)

    8. `export async function getCachedWorldMap(seed: number): Promise<Buffer | null>`:
       - `return await redis.getBuffer(\`worldmap:seed:${seed}\`)`

    9. `export async function cacheChunkHeights(seed: number, cx: number, cz: number, buf: Buffer): Promise<void>`:
       - `await redis.setBuffer(\`chunk:seed:${seed}:${cx}:${cz}\`, buf)`

    10. `export async function getCachedChunkHeights(seed: number, cx: number, cz: number): Promise<Buffer | null>`:
        - `return await redis.getBuffer(\`chunk:seed:${seed}:${cx}:${cz}\`)`

    11. `export async function getOrGenerateChunkHeights(seed: number, cx: number, cz: number, worldMap: WorldMap, perm: Uint8Array): Promise<Buffer>`:
        - Check Redis cache first
        - If miss: call generateChunkHeights(cx, cz, worldMap, perm), cache result, return
        - If hit: return cached buffer
